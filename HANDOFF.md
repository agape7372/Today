## 3. 핵심 진단: 앱 완성도 & 다중 치료사 동시 사용 위험 📋

전체 코드(~9,800 LOC) 검토 완료. **요약: 1인/1기기·키오스크 기준 완성도 높음(8.5/10). 그러나 "여러 치료사가 각자 기기로 동시 사용"하면 설계 전제가 무너짐.**

### 뿌리 원인
- **백엔드/DB 전혀 없음.** 100% 클라이언트. 모든 변경 데이터가 그 기기 브라우저 **localStorage**(`today-*-v1` 키)에만 저장. 기기 간 동기화는 **수동 JSON 내보내기/가져오기**가 유일.

### 🔴 Critical 위험 (다중 사용자)
1. **회의 보드(`/meeting`)** — 투표/댓글/작성자까지 있는 "협업" UI지만 `today-meeting-v1`로 기기 한정 저장 → **서로 다른 빈 보드를 봄. 공유 자체가 안 됨.** (가장 배신감 큰 지점)
2. **재고(`/inventory`)** — `today-inventory-v1` 기기별. "누가 이 교구 쓰는 중?" 공유 불가 → 동시 반출 못 막음.
3. **콘텐츠 편집** — `today-content-overrides-v1` 등 기기별. 둘이 같은 게임 편집 후 JSON 합치면 **blind 덮어쓰기**(`DataManager.tsx:86-122`, merge 아님) → 한쪽 작업 영구 소실.

### 🟠 데이터 안전성
- **iOS Safari ITP:** 7일 미사용 시 localStorage 자동 폐기 → PWA로 쓰다 일주일 안 들어가면 재고·회의·편집 통째 증발 가능.
- **quota 초과 조용히 무시:** 모든 `write()`가 `catch { /* quota */ }` → 저장 실패해도 경고 없음.

### 🔴 PIN 보안(`lib/pin.ts`) = 장식용
- SHA-256 해시를 localStorage 평문 저장, 서버 검증 없음(`today-pin-session-v1`에 미래시각 직접 넣으면 우회), 4자리+솔트없음, **내보내기 JSON에 PIN 해시 포함.** → 실수 방지용은 OK, 민감정보 보호 절대 불가.

### 🟢 좋은 점
- 글자크기 조절(fontScale), 다크모드, 프린트, maskable 아이콘 등 현장 접근성 배려 양호. 타입 안전·일관된 패턴.
